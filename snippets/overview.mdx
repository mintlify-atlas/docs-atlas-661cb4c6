## How collections work

Collections organize articles using:

- **Tag matchers**: Automatically include articles with specific tags
- **Language matchers**: Group articles by programming language
- **Parent collections**: Create hierarchies of related content
- **Custom descriptions**: Provide context for each collection's focus

<Warning>
  The collection structure is defined in YAML files under `content/collections`. Only modify these files if you're fixing bugs or improving existing collections.
</Warning>

## Finding the right content

Use the search functionality to find articles by:

- Article title or heading
- Tags and categories
- Programming language
- Description content
- Code examples

Each article card displays the title, description, tags and relevant metadata to help you quickly identify useful content.

## Collection metadata

Each collection includes:

- **Title**: The full name of the collection
- **Short title**: A condensed version for navigation
- **Description**: Detailed explanation of the collection's content and focus
- **Short description**: A brief summary for cards and previews
- **Tags**: Keywords that define the collection's scope
- **Splash image**: Visual identifier for the collection

This metadata helps you understand what each collection contains before browsing its articles.
